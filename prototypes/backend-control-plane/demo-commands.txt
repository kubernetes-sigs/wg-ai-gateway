Backend Demo Commands
=====================

IMPORTANT: You need a port-forward running before you can curl.
Open a dedicated terminal for it and leave it running:

  kubectl port-forward svc/envoy-httpbin-gateway 8080:80


1. CURL THE GATEWAY
--------------------
This sends a request through the Gateway -> Envoy -> httpbin.org
and returns a JSON response with your request headers/origin.

  curl -s http://localhost:8080/get | python3 -m json.tool


2. CONTROLLER LOGS
-------------------
The ai-gateway controller reconciles Gateway/HTTPRoute/Backend
resources and pushes xDS config to Envoy. Look for "Successfully synced".

  kubectl logs -n ai-gateway-system deploy/ai-gateway-controller -f


3. ENVOY PROXY LOGS
---------------------
The Envoy proxy handles the actual traffic. You'll see access logs
for each request you curl (method, path, response code, upstream).

  kubectl logs -l app=envoy-httpbin-gateway -f


4. CHECK RESOURCE STATUS
-------------------------
Quick health check. Gateway should be Programmed=True,
HTTPRoute should be Accepted=True and ResolvedRefs=True.

  kubectl get gateway,httproute,xbackenddestinations.ainetworking.prototype.x-k8s.io
